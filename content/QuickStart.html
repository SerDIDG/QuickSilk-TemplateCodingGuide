Начальная стадия

Разработка темплейтов для QuickSilk CMS осуществляется на специальном локальном сервере: http://192.168.20.101/. На данный момент он доступен только во внутриофисной сети.

Для удобства начальную стадию можно проводить и у себя на локальной машине. Вёрстку структуры, стилизацию типографии, контента и стандартных view модулей можно делать на базе пустого шаблона. Скачать его можно здесь. В состав пустого шаблона входят две библиотеки UI в которых есть все стили CMS и её модулей. Примеры некоторые элементов и модулей можно найти в документах шаблона.

  • Постоянная ссылка на пустой шаблон - https://www.dropbox.com/s/f51jtm26rvjwiod/QuickSilk%20-%2000001%20-%20Coding%20Exampe.zip?dl=0
  • docs/common.html - примеры элементов UI.
  • docs/application.html - примеры модулей и блоков CMS.
  • layout.html - пример структуры шаблона (серверный шаблон содержит ещё и специальные вставки кода).

Подготовка среды

Под каждый новый шаблон следует создавать Microsite - под сайт со своим выбранным шаблоном, разметкой (layout) и контентом. Администраторский раздел модуля Microsite находится здесь (Modules > Microsites). Форму создания Microsite следует заполнять по шаблону ниже. Имя шаблона и ID следует брать из таблицы шаблонов. В поле Template следует выбрать шаблон на базе которого будет делаться новый. Рекомендуется выбирать Blank template #1.

Обновление файлов шаблона на сервере и очистка кеша

Для того чтобы править файлы шаблона нового микросайта, следует подключиться к FTP серверу и перейти в директорию public/web/ID/templates/Template_ID, где ID - Microsite URL заполненный в форме создания микросайта, Template_ID - ID выбранного шаблона в форме создания микросайта. После того, как будут залиты на сервер новые less файлы шаблон, следует сбросить настройки шаблона с Администраторской панели. Чтобы отчистить настройки шаблона следует пройти авторизацию на микросайте и перейти в Менеджер Шаблонов (Modules > Templates). В списке шаблонов поднести курсор к выбранному при создании микросайта шаблону и нажать Customize. На открывшейся странице в каждой табе нажать кнопку Reset to default settings. Так же нужно очистить кеш ассетов, для этого переходим на страницу Settings, открываем табу Service и жмёшь по ссылке Assets files: Clear.

  • Авторизация на микросайте: http://192.168.20.101/ID/user/login, где ID - Microsite URL заполненный в форме создания микросайта.
  • Менеджер шаблонов - Templates: http://192.168.20.101/ID/admin/template-library, где ID - Microsite URL заполненный в форме создания микросайта.
  • Сервисная страница - Settings: http://192.168.20.101/ID/admin/setting, где ID - Microsite URL заполненный в форме создания микросайта.

Материалы шаблона

Все ресурсы шаблона, которые будут использоваться в качестве контента должны быть загружены через файл менеджер в специальную папку - files. Для удобства можно группировать изображения по папкам. Например для иконок создавать папку - icons, для слайдеров - slider, и тд. При сохранении изображений старайтесь удалять все метаданные.

  • Файл менеджер: http://192.168.20.101/ID/FileManager/index, где ID - Microsite URL заполненный в форме создания микросайта.

У каждого шаблона обязательно должно быть три скриншота и один эскиз. Эскиз используется в качестве картинки в Галерее Шаблонов, должен отражать суть шаблона. Скриншоты должны содержать только шаблон от неавторизованного пользователя и без рамок окон браузера. Эти файлы стоит поместить в папку resources в директории files файл менеджера.

  • Папка: File Manager/files/resources.
  • Эскиз: 640x480. 80% сжатие. thumbnail.jpg. Пример эскиза здесь.
  • Скриншоты: любой размер по высоте, но не менее 1024px, ширина - 1280px. Для скриншота newsletter ширина - 1024px. 75% сжатие. 1.jpg - 3.jpg.
